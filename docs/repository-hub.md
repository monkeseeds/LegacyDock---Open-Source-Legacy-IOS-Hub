# Legacy Repository Hub And Community Resources

LegacyDock includes a Repository Hub inside the console. Its purpose is to reduce scattered searching across Reddit, Discord, old forums, and stale package lists.

## Included Surfaces

- Searchable repository browser.
- Repository health status, last check, response notes, SSL/HTTP support, package count, category, maintainer, rating, and verification state.
- Add to Cydia links and copy URL actions.
- Browser-safe manual health refresh using the local community database.
- Package browser with version, dependency, compatibility, repository, rating, verification, and last-updated metadata.
- Essential tweaks database for utilities, recording, UI, restoration, and applications.
- Legacy service catalog for [TubeRepair](https://github.com/bag-xml/TubeRepair), [MapsX](https://cydia.skyglow.es/), Weather, iTunes Store, App Store, push notifications, Siri, Game Center, and YouTube restoration research.
- Repository submission form staged for future moderation.
- Dead repository archive with original URLs, mirror status, and recovery instructions.
- Community resource center for [r/LegacyJailbreak](https://www.reddit.com/r/LegacyJailbreak/), [iPhoneOS Obscura](https://github.com/CatsLover2006/iOSobscuraServer), [Legacy iOS Kit](https://github.com/LukeZGD/Legacy-iOS-Kit), [ChatGPT for Legacy iOS](https://github.com/bag-xml/ChatGPT-for-Legacy-iOS), [Discord Classic](https://github.com/bag-xml/Discord-Classic), and related trusted resources.
- Device-specific smart recommendations based on selected model and firmware.

LegacyDock does not own those repositories or packages. The current Hub is metadata-first: source URLs, compatibility notes, health status, service restoration guidance, and community resource links. It does not mirror package payloads or redistribute copyrighted software.

Future live repository integrations should keep maintainer credits visible, preserve source URLs, separate verified status from community reports, and provide maintainers a path to correct metadata.

## Device Doctor Integration

Device Doctor uses the repository catalog to detect stale sources, duplicate mirrors, missing recommended repositories, broken package metadata, package-manager mismatches, and compatible restoration packages.

Current GitHub Pages behavior is local and fixture-backed. A future desktop or hosted service can replace these local health values with live repository probes, moderation queues, signed community reports, and broader package-manager-aware detection.
