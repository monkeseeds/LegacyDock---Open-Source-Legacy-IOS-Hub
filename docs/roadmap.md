# Roadmap

## Phase 1: Local Product Skeleton

- Landing page and static product console.
- JSON schemas for core entities.
- Mock device, package, repository, and snapshot data.
- Compatibility engine.
- Repository scanner.
- Snapshot creation and diffing.
- Preservation report export.

## Phase 2: Native Desktop MVP

- Tauri shell.
- SQLite local store.
- Device detection adapter.
- Read-only package and repository import.
- Local snapshot persistence.
- Health scan history.

## Phase 3: Safe Management

- Repository add, remove, verify, refresh, duplicate detection, and repair.
- Dependency graph resolution.
- Install and uninstall previews.
- Restore preview from snapshot.
- Explicit risk gates for destructive actions.

## Phase 4: Community Intelligence

- Opt-in anonymous reports.
- Compatibility database.
- Package success rates by firmware and device.
- Conflict and crash intelligence.
- Recommendation engine.

## Phase 5: Cloud Convenience

- Encrypted snapshot sync.
- Shared collections.
- Favorites.
- Device history.
- Studio inventory and repair shop workspace.
- Hosted APIs for research and preservation partners.
